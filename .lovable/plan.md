

## Анализ текущего состояния и план улучшений

После детального анализа всех компонентов сайта, выявлены следующие области для улучшения:

---

### 1. SEO: обновить JSON-LD и добавить FAQPage schema

**Проблема:** В JSON-LD `streetAddress` содержит только "Туапсинский район" без точного адреса. Нет FAQPage schema — а это даёт расширенные сниппеты в Google.

**Что сделать:**
- Обновить `streetAddress` на `"ул. Кооперативная, д. 39"` в `index.html`
- Добавить второй `<script type="application/ld+json">` блок с `FAQPage` schema на основе 10 вопросов из FaqSection

---

### 2. Производительность: отключить WebGL-шейдер на мобильных

**Проблема:** `ShaderCanvas` в PricesSection запускает WebGL requestAnimationFrame loop на всех устройствах, что расходует батарею на телефонах.

**Что сделать:**
- Добавить проверку `window.innerWidth < 768` и не рендерить `<ShaderCanvas />` на мобильных
- Добавить CSS gradient fallback вместо шейдера

---

### 3. Доступность (a11y): aria-labels и фокус-стили

**Проблема:** Кнопки `−` и `+` в калькуляторе цен не имеют `aria-label`. Кнопки выбора гостей (1-5) тоже без aria-label.

**Что сделать:**
- Добавить `aria-label` на кнопки `−`/`+` в PricesSection
- Добавить `aria-label` на кнопки выбора количества гостей

---

### 4. Контакты: добавить WhatsApp и Telegram

**Проблема:** Сейчас единственный мессенджер — MAX. Многие пользователи привыкли к WhatsApp/Telegram.

**Что сделать:**
- Добавить ссылки на WhatsApp и Telegram в Footer рядом с MAX
- Добавить иконки из lucide-react (MessageCircle для WhatsApp/Telegram)

---

### 5. Footer: добавить ссылку на политику конфиденциальности

**Проблема:** Нет ссылки на политику конфиденциальности — это требование закона для сайтов, собирающих персональные данные (форма бронирования).

**Что сделать:**
- Добавить текстовую ссылку "Политика конфиденциальности" в нижнюю строку футера

---

### 6. Форма бронирования: маска телефона

**Проблема:** Поле телефона в BookingForm не форматирует ввод — пользователь может ввести что угодно.

**Что сделать:**
- Добавить простую маску/форматирование для поля телефона при вводе

---

### Файлы, которые будут изменены:
- `index.html` — JSON-LD streetAddress + FAQPage schema
- `src/components/PricesSection.tsx` — mobile check для ShaderCanvas, aria-labels
- `src/components/Footer.tsx` — WhatsApp/Telegram ссылки, ссылка на политику
- `src/components/BookingForm.tsx` — маска телефона
